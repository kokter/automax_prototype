# Поиск для Автомакса (прототип)

Это прототип поиска по товарам для магазина автозапчастей. Делал по итогам
исследования — выбрали гибридный поиск (обычный поиск по словам + поиск по смыслу
через нейросеть), всё это сделано как отдельный сервис, к которому можно
подключаться по API.

## Что умеет

- искать товары по запросу (учитывает склонения, опечатки и синонимы/сленг)
- складывать товары в индекс (POST /index)
- работать сразу с несколькими магазинами — у каждого свой API-ключ, данные не
  смешиваются
- есть простая веб-страничка, чтобы потыкать поиск руками

Внутри:
- лексический поиск — pymorphy3 (леммы) + rapidfuzz (опечатки) + BM25
- семантический поиск — модель bge-m3 + база Qdrant
- результаты двух поисков склеиваются через RRF

## Файлы

```
app/main.py            - сам сервис (эндпоинты + веб-страница)
app/config.py          - настройки (читаются из .env)
app/models.py          - схемы запросов/ответов
app/search/lexical.py  - поиск по словам
app/search/semantic.py - поиск по смыслу (bge-m3 + Qdrant)
app/search/hybrid.py   - склейка результатов (RRF)
docker-compose.yml     - запуск Qdrant
sample_catalog.json    - демо-каталог (18 товаров)
seed.py                - закидывает демо-каталог в сервис
.env.example           - пример настроек
```

## Как запустить

Нужны Python и Docker.

1. Сделать файл настроек из примера:
```
cp .env.example .env
```

2. Запустить базу Qdrant:
```
docker compose up -d
```

3. Поставить библиотеки:
```
python -m venv .venv
source .venv/bin/activate      # на Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

4. Запустить сервис (первый запуск скачает модель bge-m3, примерно 2 ГБ, это долго):
```
uvicorn app.main:app --reload
```

5. Закинуть демо-товары:
```
python seed.py
```

6. Открыть в браузере http://localhost:8000 и попробовать запросы:
`граната`, `незамерзайка`, `помпа`, `аммортизатор`.

## Примеры запросов

Положить каталог:
```
curl -X POST http://localhost:8000/index \
  -H "X-API-Key: demo-key-automax" -H "Content-Type: application/json" \
  -d @sample_catalog.json
```

Найти товар:
```
curl "http://localhost:8000/search?q=граната&k=5" -H "X-API-Key: demo-key-automax"
```

Ключ `demo-key-automax` — это магазин automax. Можно завести второй магазин другим
ключом, товары у них будут отдельно.

## Настройки (.env)

- QDRANT_URL — где база Qdrant (по умолчанию http://localhost:6333)
- EMBED_MODEL и VECTOR_SIZE — модель и размер вектора (для bge-m3 это 1024)
- API_KEYS — ключи магазинов в виде ключ:магазин через запятую
- CANDIDATES — сколько кандидатов брать из каждого поиска
- RRF_K — настройка склейки RRF
- PROTOTYPE_FAKE_EMBED — поставить 1, чтобы проверить сервис без скачивания модели
  (поиск по смыслу при этом не работает, только для теста что всё запускается)
